Code organization

First, let's take a look at the Spirodraw declarations.

{% prettify dart %} library spirodraw; import 'dart:html'; import 'dart:math' as Math; part "ColorPicker.dart"; {% endprettify %}

A Dart script belongs to a library declared with a name given by the library directive. Dart scripts include other scripts using the part directive. Dart libraries can also use other libraries with the import directive.

Spirodraw.dart defines a library named 'spirodraw', which contains Spirodraw.dart and ColorPicker.dart, and imports the built-in Dart DOM library.

requestAnimationFrame

Spirodraw uses requestAnimationFrame to improve rendering.

{% prettify dart %} void start() { refresh(); rad = 0.0; run = true; window.requestAnimationFrame(animate, frontCanvas); } void animate(int time) { if (run && rad <= maxTurns * PI2) { rad+=stepSize; drawFrame(rad); num nTurns = rad / PI2; numTurns.innerText = '${nTurns.floor()}/$maxTurns'; window.webkitRequestAnimationFrame(animate, frontCanvas); } else { stop(); } } {% endprettify %}

In the code excerpt above, line 5 invokes requestAnimationFrame with a closure containing the callback function, animate(). The animate() function in turn requests the next animation frame, passing a reference to itself as the callback.

String interpolation

As shown in line 13 above, you can evaluate variables in a string literal using $identifier or ${expression}. This syntax offers a shortcut for string concatenation using the + operator.

Top-level functions

In Dart, a function does not have to be associated with a class.

{% prettify dart %} ... void main() { new Spirodraw().go(); } class Spirodraw { ... int calcTurns() { // compute ratio of wheel radius to big R then find LCM if ((dUnits==0) || (rUnits==0)) return 1; int ru = rUnits.abs(); int wrUnits = RUnits + rUnits; int g = gcf (wrUnits, ru); return (ru ~/ g).toInt(); } ... } int gcf(int n, int d) { if (n==d) return n; int max = Math.max(n, d); for (int i = max ~/ 2; i > 1; i--) if ((n % i == 0) && (d % i == 0)) return i; return 1; } {% endprettify %}

In this example, note that the gcf() function is defined outside the Spirodraw class and is invoked within the calcTurns() function. The main() function is likewise defined outside the Spirodraw class. In Dart, variable declarations can also be top-level, but not statements or expressions.

ColorPicker sample

Now let's look at ColorPicker.dart.

{% prettify dart %} part of spirodraw; typedef void PickerListener(String selectedColor); class ColorPicker { static final hexValues = const ['00', '33', '66', '99', 'CC', 'FF']; static final COLS = 18; // Block height, width, padding static final BH = 10; static final BW = 10; static final BP = 1; final List _listeners; CanvasElement canvasElement; String _selectedColor = 'red'; final height = 160; final width = 180; CanvasRenderingContext2D ctx; ColorPicker(this.canvasElement) : _listeners = [] { ctx = canvasElement.context2d; drawPalette(); addHandlers(); showSelected(); } String get selectedColor() => _selectedColor; void set selectedColor(String color) { _selectedColor = color; showSelected(); fireSelected(); } void onMouseMove(MouseEvent event) { int x = event.offsetX; int y = event.offsetY - 40; if (( y < 0) || (x >= width)) { return; } ctx.fillStyle = getHexString(getColorIndex(x, y)); ctx.fillRect(0, 0, width/2, 30); } void onMouseDown(MouseEvent event) { Element elt = event.target; event.cancelBubble = true; int x = event.offsetX; int y = event.offsetY - 40; if ((y < 0) || (x >= width)) return; selectedColor = getHexString(getColorIndex(x, y)); } /** * Adds a [PickerListener] to receive updates. */ void addListener(PickerListener listener) { _listeners.add(listener); } void addHandlers() { canvasElement.on.mouseMove.add((e) => onMouseMove(e), true); canvasElement.on.mouseDown.add((e) => onMouseDown(e), true); } void drawPalette() { int i=0; for (int r=0; r < 256; r+=51) { for (int g=0; g < 256; g+=51) { for (int b=0; b < 256; b+=51) { String color = getHexString(i); ctx.fillStyle = color; int x = BW * (i % COLS); int y = BH * (i ~/ COLS) + 40; ctx.fillRect(x + BP, y + BP, BW - 2 * BP, BH - 2 * BP); i++; } } } } void fireSelected() { for (final listener in _listeners) { listener(_selectedColor); } } int getColorIndex(int x, int y) { // Get color index 0-215 using row, col int i = y ~/ BH * COLS + x ~/ BW; return i; } void showSelected() { ctx.fillStyle = _selectedColor; ctx.fillRect(width / 2, 0, width / 2, 30); ctx.fillStyle = "white"; ctx.fillRect(0, 0, width / 2, 30); } String getHexString(num value) { int i = value.floor().toInt(); int r = (i ~/ 36) % 6; int g = (i % 36) ~/ 6; int b = i % 6; return '#${hexValues[r]}${hexValues[g]}${hexValues[b]}'; } } {% endprettify %}

Function type aliases

In Dart, functions are objects just like strings and numbers. If you want to use a function as a variable type or return type, you can use a function type alias, or typedef.

In the example above, the typedef statement declares a function type PickerListener that takes a String argument and returns void.

Function type aliases offer a compact way to represent any type of callback.

Type propagation

Dart types are automatically propagated from constants and literals. For example, the line _listeners = [] initializes a List of Strings, but the variable type List<String> is omitted because the type can be determined from the string literals.

Constructor initializers

Often constructor arguments are used to initialize instance fields. To avoid repetitive this.name = name initializers, Dart allows you to write this.name in the constructor instead of the argument name. See the ColorPicker constructor for an example. Also, Dart lets you write an initializer list in the constructor. Initializers are executed before the instance is constructed and are used to initialize final fields.

Terse function syntax

Functions that simply return an expression can be declared in a compact one-line syntax using => as shown with selectedColor() getter.

Getters and setters

Note the definition of an explicit getter and setter for selectedColor. When you define a getter or setter function this way, Dart will automatically invoke the getter or setter when you reference a variable having the same name as the getter/setter. By convention, the actual variable name is prefixed with an underscore, which makes it private to the library and distinguishes it from the getter/setter name.

For-in statement

Our last bit of syntactic sugar is shown with for (final listener in _listeners). The for-in loop iterates over any object that implements the Iterable interface.
